# <a name="endpoint-calls-faulted"></a>Конечная точка: сбои вызовов

Имя счетчика: Calls Faulted  
  
## <a name="description"></a>Описание  

 Количество вызовов данной конечной точки, которые возвратили сбой. В приложениях Windows Communication Foundation (WCF) методы служб обмениваются сведениями об ошибках обработки с помощью сообщений о сбоях SOAP. Сообщения об ошибках SOAP - это типы сообщений, которые включаются в метаданные, связанные с операцией службы, и таким образом создают контракт ошибок, который клиенты могут использовать для повышения надежности и интерактивности своей работы. Поскольку сообщения об ошибках SOAP передаются клиентам в формате XML, они поддерживают возможность взаимодействия.

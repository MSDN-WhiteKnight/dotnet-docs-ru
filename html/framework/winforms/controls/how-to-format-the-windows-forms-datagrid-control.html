Практическое руководство. Форматирование элемента управления DataGrid в Windows Forms

Note

Элемент управления DataGridView заменяет элемент управления DataGrid и расширяет его функциональные возможности; однако при необходимости элемент управления DataGrid можно сохранить для обратной совместимости и использования в будущем. Дополнительные сведения см. в разделе Различия элементов управления DataGridView и DataGrid в Windows Forms.

Разными цветами на различные части DataGrid управления позволяет облегчить чтение и понимание сведений. Цвет могут применяться в строки и столбцы. Строки и столбцы можно также скрывать или отображать по необходимости.

Существует три основных аспектов форматирования DataGrid элемента управления. Можно задать свойства, чтобы создать стиль по умолчанию, в котором отображаются данные. На этой основе можно настроить способ отображения определенных таблиц во время выполнения. Наконец можно изменить столбцы, которые отображаются в сетке данных, а также цвета и другие элементы, которые отображается.

На первом этапе форматирования сетки данных, можно задать свойства DataGrid сам. Эти параметры цветов и форматирования составляют основу, из которого затем можно внести изменения в зависимости от данных таблиц и столбцов, отображаемых.

Чтобы создать стиль по умолчанию для элемента управления DataGrid

  1. Задайте следующие свойства соответствующим образом:

    Свойство Описание
    AlternatingBackColor BackColor Свойство определяет цвет четных строк сетки. При задании AlternatingBackColor цвет, все остальные строки свойству цветом (строки 1, 3, 5 и т. д.).
    BackColor Цвет фона четных строк сетки (строки 0, 2, 4, 6 и т. д.).
    BackgroundColor Тогда как BackColor и AlternatingBackColor свойства определяет цвет строк в сетке BackgroundColor свойство определяет цвет области незанятой строками, которая отображается, только когда сетка прокручена вниз, а также если только несколько строк, содержащихся в Сетка.
    BorderStyle Стиль границы сетки, один из BorderStyle значений перечисления.
    CaptionBackColor Цвет фона заголовка сетки окна, расположенного непосредственно над сеткой.
    CaptionFont Шрифт заголовка в верхней части сетки.
    CaptionForeColor Цвет фона названия окна сетки.
    Font Шрифт, используемый для отображения текста в сетке.
    ForeColor Цвет шрифта, которым данные в строках таблицы данных.
    GridLineColor Цвет линий сетки сетки данных.
    GridLineStyle Стиль линий, разделяющих ячейки сетки, один из DataGridLineStyle значений перечисления.
    HeaderBackColor Цвет фона заголовков строк и столбцов.
    HeaderFont Шрифт, используемый для заголовков столбцов.
    HeaderForeColor Основной цвет заголовков столбцов сетки, включая текст заголовка столбца и глифы плюс/минус (для Развернуть строки, при отображении нескольких связанных таблиц).
    LinkColor Цвет текста всех ссылок в сетке данных, включая ссылки на дочерние таблицы, имя отношения и т. д.
    ParentRowsBackColor В дочерней таблице это цвет фона родительских строк.
    ParentRowsForeColor В дочерней таблице это основной цвет родительских строк.
    ParentRowsLabelStyle Определяет, отображаются ли имена таблиц и столбцов в родительской строке, с помощью параметра DataGridParentRowsLabelStyle перечисления.
    PreferredColumnWidth Стандартная ширина (в пикселях) столбцов сетки. Задать это свойство перед сбросом DataSource и DataMember свойства (либо отдельно, либо с помощью SetDataBinding метод), или свойство не окажет никакого воздействия.

    Свойство не может быть присвоено значение меньше 0.
    PreferredRowHeight Высота строки (в пикселях) строки в сетке. Задать это свойство перед сбросом DataSource и DataMember свойства (либо отдельно, либо с помощью SetDataBinding метод), или свойство не окажет никакого воздействия.

    Свойство не может быть присвоено значение меньше 0.
    RowHeaderWidth Ширина заголовков строк сетки.
    SelectionBackColor При выборе ячейки или строки, это цвет фона.
    SelectionForeColor При выборе ячейки или строки, это цвет переднего плана.
    Note

    Имейте в виду, при настройке цветов элементов управления, что можно сделать элемент управления недоступен из-за выбор неудачных цветов (например, красный и зеленый). Использование цветов, доступных на системных цветов палитры модулей, чтобы избежать этой проблемы.

    В следующих процедурах предполагается, в форме есть DataGrid элемент управления привязан к таблице данных. Дополнительные сведения см. в разделе привязка элемента управления DataGrid в Windows Forms к источнику данных.

Если требуется задать стиль таблицы и столбца таблицы данных программным способом

  1. Создать новый стиль таблицы и задать его свойства.

  2. Создайте стиль столбца и задайте его свойства.

  3. Добавьте стиль столбца в коллекцию стилей столбцов стиль таблицы.

  4. Добавьте стиль таблицы в коллекцию стилей таблиц в сетке данных.

  5. В следующем примере, создайте экземпляр нового DataGridTableStyle и задайте его MappingName свойство.

  6. Создайте новый экземпляр класса GridColumnStyle и задайте его MappingName (и некоторые другие свойства отображения и макета).

  7. Повторите шаги 2 – 6 для каждого стиля столбца, который вы хотите создать.

    В следующем примере показано как DataGridTextBoxColumn создается, поскольку имя будет отображаться в столбце. Кроме того, добавьте стиль столбца GridColumnStylesCollection стиля таблицы, и добавьте стиль таблицы для GridTableStylesCollection сетки данных.

    Private Sub CreateAuthorFirstNameColumn()  
   ' Add a GridTableStyle and set the MappingName   
   ' to the name of the DataTable.  
   Dim TSAuthors As New DataGridTableStyle()  
   TSAuthors.MappingName = "Authors"  

   ' Add a GridColumnStyle and set the MappingName   
   ' to the name of a DataColumn in the DataTable.   
   ' Set the HeaderText and Width properties.   
   Dim TCFirstName As New DataGridTextBoxColumn()  
   TCFirstName.MappingName = "AV_FName"  
   TCFirstName.HeaderText = "First Name"  
   TCFirstName.Width = 75  
   TSAuthors.GridColumnStyles.Add(TCFirstName)  

   ' Add the DataGridTableStyle instance to   
   ' the GridTableStylesCollection.   
   myDataGrid.TableStyles.Add(TSAuthors)  
End Sub  

    private void addCustomDataTableStyle()  
{  
   // Add a GridTableStyle and set the MappingName   
   // to the name of the DataTable.  
   DataGridTableStyle TSAuthors = new DataGridTableStyle();  
   TSAuthors.MappingName = "Authors";  

   // Add a GridColumnStyle and set the MappingName   
   // to the name of a DataColumn in the DataTable.   
   // Set the HeaderText and Width properties.   
   DataGridColumnStyle TCFirstName = new DataGridTextBoxColumn();  
   TCFirstName.MappingName = " AV_FName";  
   TCFirstName.HeaderText = "First Name";  
   TCFirstName.Width = 75;  
   TSAuthors.GridColumnStyles.Add(TCFirstName);  

   // Add the DataGridTableStyle instance to   
   // the GridTableStylesCollection.   
   dataGrid1.TableStyles.Add(TSAuthors);  
}  

    private:  
   void addCustomDataTableStyle()  
   {  
      // Add a GridTableStyle and set the MappingName   
      // to the name of the DataTable.  
      DataGridTableStyle^ TSAuthors = new DataGridTableStyle();  
      TSAuthors->MappingName = "Authors";  

      // Add a GridColumnStyle and set the MappingName   
      // to the name of a DataColumn in the DataTable.   
      // Set the HeaderText and Width properties.   
      DataGridColumnStyle^ TCFirstName = gcnew DataGridTextBoxColumn();  
      TCFirstName->MappingName = "AV_FName";  
      TCFirstName->HeaderText = "First Name";  
      TCFirstName->Width = 75;  
      TSAuthors->GridColumnStyles->Add(TCFirstName);  

      // Add the DataGridTableStyle instance to   
      // the GridTableStylesCollection.   
      dataGrid1->TableStyles->Add(TSAuthors);  
   }

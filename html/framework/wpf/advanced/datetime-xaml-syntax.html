Синтаксис DateTime XAML

Некоторые элементы управления, такие как Calendar и DatePicker, имеют свойства, использующие DateTime типа. Несмотря на то что начальные дата и время для этих элементов управления обычно указываются в коде программной части во время выполнения, их можно указать в коде XAML. Средство синтаксического анализа XAML WPF обрабатывает анализ DateTime значения с помощью встроенных текстового синтаксиса XAML. В этом разделе описаны особенности DateTime синтаксис текста XAML.

Когда требуется использовать синтаксис DateTime XAML

Задание дат в XAML не всегда обязательно, а иногда даже может быть нежелательно. Например, можно использовать DateTime.Now свойство для инициализации даты во время выполнения, или сделать все корректировки дат для календаря в коде программной части на основе ввода пользователя. Однако существуют сценарии, где можно жестко закодировать даты в Calendar и DatePicker в шаблоне элемента управления. DateTime Необходимо использовать синтаксис XAML для этих сценариев.

Синтаксис DateTime XAML является собственным поведением

DateTime — Это класс, который определен в библиотеках базовых классов среды CLR. Из-за связи библиотеки базовых классов с остальной части среды CLR, он уже не сможете применить TypeConverterAttribute класс и использовать преобразователь типов для обработки строк из XAML и преобразовать их DateTime в объектной модели времени выполнения. Нет класса DateTimeConverter, предоставляющего поведение преобразования: поведение преобразования, описанное в этом разделе, является собственным для средства синтаксического анализа XAML WPF.

Строки формата для синтаксиса DateTime XAML

Можно указать формат DateTime строку формата. Строки формата формализуют синтаксис текста, который может использоваться для создания значения. DateTime значения для существующие элементы управления WPF обычно только используют компоненты даты DateTime и не компоненты времени.

При указании DateTime в XAML, можно использовать любой из этих строк формата взаимозаменяемо.

Можно также использовать форматы и строки формата, которые не рассматриваются в этом разделе. С технической точки зрения XAML для любого DateTime значение, которое указывается и затем анализируется средством синтаксического анализа XAML WPF использует внутренний вызов DateTime.Parse, таким образом можно использовать любой строкой, принимаемое DateTime.Parse вашей XAML ввода. Дополнительные сведения см. в разделе DateTime.Parse.

Important

Синтаксис DateTime XAML всегда использует en-us как CultureInfo для собственного преобразования. Это поведение не зависит от Language значение или xml:lang значение в XAML, так как преобразование типов на уровне атрибутов XAML работает без этого контекста. Не пытайтесь выполнять интерполяцию показанных здесь строк формата из-за различий региональных параметров, таких как порядок отображения дня и месяца. Показанные здесь строки формата — это именно те строки формата, которые используются при синтаксическом анализе XAML независимо от других параметров языка и региональных параметров.

В следующих разделах описаны некоторые распространенные DateTime форматирования строк.

Шаблон краткого формата даты (d)

Ниже приведен краткий формат даты для DateTime в XAML:

M/d/YYYY

Это самая простая форма, определяющая все необходимые сведения для типичного использования элементами управления WPF. На нее не влияют случайные смещения часового пояса (по сравнению с компонентом времени), и поэтому она предпочтительнее других форматов.

Например, чтобы указать дату 1 июня 2010 г., используйте следующую строку.

3/1/2010

Дополнительные сведения см. в разделе DateTimeFormatInfo.ShortDatePattern.

Сортируемый шаблон даты и времени (s)

Далее показано, что данные в элементе DateTime шаблон в XAML:

yyyy'-'MM'-'dd'T'HH':'mm':'ss

Например, чтобы указать дату 1 июня 2010 г., используйте следующую строку (все компоненты времени вводятся как 0).

2010-06-01T000:00:00

Шаблон RFC1123 (r)

Шаблон RFC1123 удобен, поскольку он может быть строкой ввода из других генераторов данных, также использующих шаблон RFC1123, в ситуациях инвариантных значений языка и региональных параметров. Ниже показан шаблон RFC1123 DateTime шаблон в XAML:

ddd, dd MMM yyyy HH':'mm':'ss 'UTC'

Например, чтобы указать дату 1 июня 2010 г., используйте следующую строку (все компоненты времени вводятся как 0).

Mon, 01 Jun 2010 00:00:00 UTC

Другие форматы и шаблоны

Как уже говорилось ранее, DateTime в XAML может быть указан как любая строка, которая является приемлемой как входные для DateTime.Parse. Сюда входят другие формализованные форматы (например UniversalSortableDateTimePattern) и форматы, которые не формализованы как определенный DateTimeFormatInfo формы. Например, в форме YYYY/mm/dd допустимо как входные для DateTime.Parse. Этот раздел не является попыткой описать все возможные форматы: вместо этого рекомендуется использование шаблона короткого формата даты.
